netmeta-verify MCP Server

Mathematical verification of network meta-analysis results and reproducible R code generation. Verify that NMA output is the unique MLE / satisfies REML conditions — without re-running the analysis.

Self-hosted installation

Option A — Docker (recommended)

The verify image additionally installs igraph and multiarmvars.

git clone --recurse-submodules https://github.com/tpapak/mcp.git
cd mcp/netmeta
docker compose up -d   # starts both netmeta (8000) and verify (8001)

Option B — Native

# 1. Follow the netmeta-mcp native install steps first, then:

conda activate netmeta-mcp

# 2. Install extra R packages
R -e "install.packages('igraph')"
R -e "remotes::install_github('tpapak/multiarmvars')"

# 3. Run the verify server (stdio)
netmeta-verify-mcp

# 3. Or HTTP
python -m netmeta_verify.http_server
# → http://localhost:8001/mcp
The verifier R scripts are in the verifier/ sibling directory. If you cloned only mcp-netmeta, set the environment variable:

export VERIFIERS_DIR=/path/to/mcp/verifier

Tools

verify_fixed_effect_mle
Verify that a fixed-effect NMA league table is the unique maximum likelihood estimate. Checks score equations, positive-definite Hessian, uniqueness, WLS equivalence, and multi-arm variance consistency.
verify_random_effect_reml
Verify that a random-effects NMA satisfies all REML optimality conditions: fixed-effects score, REML score for τ², positive-definite information matrix, τ² ≥ 0, uniqueness, GLS equivalence, and log-likelihood maximum.
generate_nma_code
Generate a complete, standalone R script that reproduces any NMA. Embeds data, runs netmeta(), and optionally includes rankings, league table, forest plot, and network graph. Suitable for publication supplementary materials.

Input format for verify tools

Both verify tools accept the same pairwise contrast format as runnetmeta, plus a league_table argument:

Theory: fixed-effect MLE verification

For a fixed-effect NMA with normal contrast-level data, the maximum likelihood estimate is equivalent to weighted least squares. The verifier checks five necessary and sufficient conditions without re-fitting the model.

1

Score equations = 0

Gradient of the log-likelihood is zero at the reported estimates.

2

Information matrix positive definite

All eigenvalues of X′WX are positive, confirming a maximum not a saddle point.

3

Unique solution (full rank)

rank(X′WX) = number of parameters. Guaranteed when the network is connected.

4

Matches WLS closed form

Estimates equal (X′WX)⁻¹ X′Wy to within numerical tolerance.

5

Multi-arm variance consistency

For multi-arm studies, contrast variances must be decomposable into non-negative arm variances. Checked via the multiarmvars package.

score: X′W(y − Xθ̂) = 0 info: X′WX ≻ 0 WLS: θ̂ = (X′WX)−¹ X′Wy

Where X is the design matrix, W = V⁻¹ is the weight matrix (inverse of the within-study covariance), y are the observed contrasts, and θ̂ are the reported treatment effects.

Example output

=== Fixed-Effect NMA MLE Verification === Observations: 28  |  Studies: 26 (1 multi-arm)  |  Parameters: 9 Reference: plac  |  Tolerance: 1.00e-06 [PASS] Score equations = 0 [PASS] Information matrix positive definite [PASS] Unique solution [PASS] Matches WLS solution [PASS] Multi-arm variance consistency Result: 5/5 tests passed CONCLUSION: Output is the UNIQUE maximum likelihood estimate.

Theory: random-effect REML verification

REML estimation has two coupled optimality conditions: one for the treatment effects θ given τ², and one for the heterogeneity variance τ² itself. The verifier checks seven conditions.

1

Fixed-effects score = 0 given τ²

X′W(τ²)(y − Xθ̂) = 0

2

REML score for τ² = 0

−½[tr(P) − r′P²r] = 0 at the interior solution. At the boundary τ² = 0, the score must be ≤ 0.

3

Information matrix positive definite

Confirms a maximum in treatment-effect space.

4

τ² ≥ 0

The between-study variance is non-negative.

5

Unique solution

Network is connected, full rank.

6

Matches GLS closed form

θ̂ = (X′W(τ²)X)⁻¹ X′W(τ²)y

7

REML log-likelihood is at a maximum

Perturbation test: the REML log-likelihood decreases when τ² is shifted by ±δ.

The random-effects weight matrix is:

W(τ²) = (V + τ²I)−¹

where V is the within-study covariance matrix (diagonal for two-arm studies, with off-diagonal elements for multi-arm studies).

Example output

=== Random-Effect NMA REML Verification === tau²: 0.018762  (tau = 0.1370)  |  Tolerance: 1.00e-04 [PASS] Fixed effects score = 0 [PASS] REML score for tau² = 0 [PASS] Information matrix positive definite [PASS] tau² ≥ 0 [PASS] Unique solution [PASS] Matches GLS solution [PASS] REML is at maximum Result: 7/7 tests passed CONCLUSION: Output satisfies REML conditions.

Multi-arm study handling

Multi-arm studies contribute correlated observations. For a comparison dij between arms i and j:

Var(dij) = Vi + Vj Cov(dij, dik) = Vi (shared arm i)

The verifier uses the multiarmvars package to recover arm variances Vi from the observed contrast variances, then builds the full covariance matrix with correct off-diagonal elements.

Note on netmeta’s approximation. The netmeta package uses an approximate method that inflates variances to achieve approximate independence between contrasts. This is valid and commonly used, but gives slightly different estimates from the exact MLE. When verifying netmeta output with multi-arm data the fixed-effect MLE test will fail; the discrepancy is typically small (< 0.05).

Treatments not connected to multi-arm studies will match the exact MLE exactly. The verify tool reports per-study arm variance diagnostics so you can identify where discrepancies originate.

Reproducible R code generation

generate_nma_code produces a self-contained R script that:

  • Embeds the full dataset as inline R code (no external files)
  • Calls netmeta() with all relevant parameters
  • Optionally includes P-score rankings, league table, forest plot, network graph
  • Prints heterogeneity and Q-statistics for basic verification

Running the generated script reproduces the original results to machine precision. It is suitable for inclusion in supplementary materials or journal submissions that require reproducible analyses.

Direct R usage (without MCP)

The verifier scripts can be used standalone in any R session.

Fixed-effect verification

source("verifier/verify_fixed_effect_mle.R")

library(netmeta)
data(Senn2013)

nma <- netmeta(
  TE = Senn2013$TE, seTE = Senn2013$seTE,
  treat1 = Senn2013$treat1, treat2 = Senn2013$treat2,
  studlab = Senn2013$studlab,
  sm = "MD", common = TRUE, random = FALSE,
  reference.group = "plac"
)

v <- verify_netmeta_mle(nma)
print_verification(v)

Random-effect (REML) verification

source("verifier/verify_random_effect_reml.R")

nma <- netmeta(
  TE = Senn2013$TE, seTE = Senn2013$seTE,
  treat1 = Senn2013$treat1, treat2 = Senn2013$treat2,
  studlab = Senn2013$studlab,
  sm = "MD", common = FALSE, random = TRUE,
  method.tau = "REML", reference.group = "plac"
)

v <- verify_netmeta_reml(nma, tol = 1e-4)
print_reml_verification(v)

Verify any software output (not just netmeta)

source("verifier/verify_fixed_effect_mle.R")

# league_table: square matrix, rownames = colnames = treatment names
league_table <- matrix(
  c( 0, -0.5, -0.8,
     0.5,  0, -0.3,
     0.8,  0.3,  0),
  nrow = 3, byrow = TRUE,
  dimnames = list(c("A","B","C"), c("A","B","C"))
)

v <- verify_fixed_effect_mle(
  TE      = c(0.5, 0.8, 0.3),
  seTE    = c(0.1, 0.15, 0.12),
  treat1  = c("A","A","B"),
  treat2  = c("B","C","C"),
  studlab = c("S1","S2","S3"),
  league_table = league_table,
  reference = "C"
)
print_verification(v)

The Dockerfile.verify in mcp/netmeta/ builds the verify image. It clones this verifier/ repo into /opt/mcp/verifier at build time.

docker build -f Dockerfile.verify -t netmeta-verify-mcp .
docker run -p 8001:8001 netmeta-verify-mcp
